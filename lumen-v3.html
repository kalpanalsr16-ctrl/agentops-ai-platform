Batch vs real-time
Real-time (stream)
Every request → anomaly detection → P0 trigger. Latency <100ms. Safety flags, latency SLA. Does not run full eval — too expensive per request.
Batch (scheduled)
Weekly eval runs (Sunday 2am) + daily regression checks. 10% traffic sample after 30-day full-log window.
Why not real-time eval?
Running 50 eval prompts per request costs ~$0.40/req and adds 3–4s latency. Batch costs ~$0.002/agent/week. Real-time catches severity, batch catches depth.
Product Document
PRD & Product Thinking
Core decisions, tradeoffs, failure modes, and rollout strategy.
Product definition
One-line pitch
"The system that tells you an AI agent is going to fail — before it does, and exactly why when it does."
What this is NOT
— Not a traditional monitoring tool
— Not a logging dashboard (logs are inputs, not the product)
— Not an ML training platform
— Not a developer portal
Key tradeoffs
Speed vs quality in eval
Batch weekly vs real-time. $0.002/agent/week vs $0.40/request. Gap: new hallucination midweek caught on Sunday. Accepted at current scale.
Recall vs precision in risk scoring
High recall (94%) over precision (78%). False negative costs 10× a false positive. 22% false positive rate accepted.
Automation vs human review
Automate 80% with deterministic pass/fail. Human review for confidence <70% and all safety flags.

PHASE 2 · Q4 2026

Risk Simulator + Eval across all categories. Automate weekly evals. Begin loops 1+2. Success: rejection −15pp.
PHASE 3 · Q1 2027
Full adoption
Risk Simulator gates all publish decisions. Loop 3 quarterly. Success: rejection <15%, zero P0/quarter.

Data Schema
6 core entities. All prototype data matches these contracts.
Agent
{ id: string, name: string, category: string, version: string, developer_id: string, declared_capabilities: string[], manifest_completeness: number, // 0–1 published_at: ISO8601, health_status: "healthy"|"degrading"|"drifted"|"critical" }
EvalRun
{ agent_id: string, run_type: "pre_publish"|"weekly"|"regression"|"ab", triggered_by: "manual"|"scheduled"|"model_update", functional_score: number, safety_score: number, quality_score: number, ops_score: number, pass_fail: boolean, flagged_prompts: string[] }
RiskScore
{ agent_id: string, intent_overlap: number, // weight 30% safety_eval: number, // weight 25% accuracy_eval: number, // weight 20% latency_probe: number, // weight 15% scope_complete: number, // weight 10% composite_risk: number, recommendation: "approve"|"conditional"|"block", routing_conflicts: AgentId[], confidence: number }
ObservabilityEvent
{ agent_id: string, timestamp: ISO8601, latency_ms: number, capability_declared: string, capability_routed: string, // may differ user_outcome: "success"|"thumbs_down"|"reprompt"|"exit", model_version: string, safety_flags: string[], anomaly_flags: string[] }
FailureAttribution
{ agent_id: string, accuracy_before: number, accuracy_after: number, attributed_cause: "model_change"|"prompt_shift" |"routing_conflict"|"staleness", confidence: number, recommended_action: string, resolved_at: ISO8601 | null }
FeedbackItem
{ agent_id: string, source: "thumbs_down"|"reprompt"|"human_review", failure_type: "hallucination"|"out_of_scope" |"wrong_answer"|"safety", converted_to_test_case: boolean, loop_type: "immediate"|"weekly"|"structural" }
